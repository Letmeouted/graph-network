# 关系网络图生成器

一个基于 Web 的 Excel 数据驱动的关系网络图可视化工具，使用 ECharts 实现强大的网络图绘制和配置功能。

## 功能特性

### 核心功能
- **Excel 数据导入**: 上传包含 `node`（节点）和 `link`（连线）两个 Sheet 的 Excel 文件
- **灵活字段映射**: 支持自定义节点 ID、UID、名称、标签、分类、大小等字段
- **连线匹配**: 支持使用名称、ID 或 UID 三种方式匹配连线的源节点和目标节点
- **实时预览**: 上传数据后立即生成网络图可视化
- **属性预览**: 显示节点和连线的所有可用属性字段

### 可视化配置

#### 节点配置
| 配置项 | 说明 |
|--------|------|
| 节点 ID 字段 | 可选，未选择时自动生成索引 ID |
| 节点 UID 字段 | 可选，用于特殊场景的节点标识 |
| 节点名称字段 | 必填，作为节点的主要标识 |
| 节点标签字段 | 可选，选择要在节点上显示的字段内容（如职称、单位等） |
| 节点名称颜色 | 设置节点标签文字的颜色 |
| 标签显示位置 | 上方、下方、左侧、右侧、内部 |
| 节点透明度 | 10% - 100% 可调 |
| 节点分类字段 | 可选，用于节点分组和颜色区分 |
| 节点大小字段 | 可选，根据数值设置节点大小 |
| 默认节点大小 | 5px - 50px |

#### 连线配置
| 配置项 | 说明 |
|--------|------|
| 源节点字段 | 连线起点的字段名 |
| 目标节点字段 | 连线终点的字段名 |
| 连线匹配方式 | 使用名称/ID/UID 匹配节点 |
| 连线标签字段 | 可选，显示在连线上的文字 |
| 连线权重字段 | 可选，根据数值调整连线粗细 |

#### 显示效果
| 配置项 | 说明 |
|--------|------|
| 布局类型 | 力引导布局 / 环形布局 |
| 节点可拖拽 | 开启/关闭节点拖拽功能 |
| 标签显示比例 | 1% - 100% 的节点显示标签 |
| 标签字体大小 | 8px - 20px |
| 连线颜色 | 自定义连线颜色 |
| 图例字体颜色 | 自定义分类图例文字颜色 |
| 排斥力强度 | 影响力引导布局的节点间距 |
| 连线长度 | 影响力引导布局的连线距离 |
| 显示箭头 | 开启/关闭连线箭头 |
| 背景颜色 | 自定义图表背景色 |

#### 分类颜色配置
- 选择节点分类字段后自动显示所有分类
- 每个分类可单独设置颜色
- 支持 20+ 种默认颜色

### 导出功能
- **导出 JSON 数据**: 将处理后的节点和连线数据导出为 JSON 文件
- **导出 HTML 文件**: 生成包含当前配置和数据的独立 HTML 文件，可离线使用

## 使用说明

### 1. 准备 Excel 文件

Excel 文件需要包含两个 Sheet：

#### Sheet 1: `node` (节点数据)
| id | uid | name | category | size | 其他属性... |
|----|-----|------|----------|------|------------|
| 0 | 6504217426 | 张三 | 教授 | 25 | 单位，年龄... |
| 1 | 6504217427 | 李四 | 副教授 | 20 | 单位，年龄... |

#### Sheet 2: `link` (连线数据)
| source | target | label | weight | 其他属性... |
|--------|--------|-------|--------|------------|
| 张三 | 李四 | 合作 | 5 | 关系类型... |
| 0 | 1 | 师生 | 3 | 关系类型... |

**注意**:
- `source` 和 `target` 的值需要与节点中选择的匹配字段（名称/ID/UID）对应
- 字段名称可以自定义，上传后在配置中选择对应字段即可

### 2. 上传文件
1. 点击页面顶部的「📁 选择 Excel 文件」按钮
2. 选择准备好的 Excel 文件

### 3. 配置字段映射
1. 在左侧面板选择对应的字段
2. 节点名称字段为必填项
3. 其他字段根据需求选择

### 4. 调整显示效果
1. 选择布局类型（推荐力引导布局）
2. 调整节点大小、颜色、标签位置等
3. 如有分类，可自定义每个分类的颜色

### 5. 导出结果
- 点击「导出 JSON 数据」保存数据
- 点击「导出关系图 HTML」生成可分享的独立文件

## 技术架构

- **前端框架**: 纯原生 JavaScript
- **可视化库**: [ECharts 5.4.3](https://echarts.apache.org/)
- **Excel 解析**: [XLSX 0.18.5](https://sheetjs.com/)
- **UI 设计**: 现代渐变风格，响应式布局

## 使用示例

上传 `example.xlsx` 后，建议配置：
- **节点 ID 字段**: id
- **节点 UID 字段**: uid
- **节点名称字段**: name
- **节点标签字段**: institution（显示单位）或 title（显示职称）
- **节点分类字段**: category（按职称分类）
- **节点大小字段**: size

- **连线权重字段**: weight

## 示例场景

### 学术合作网络
- 节点：研究人员（姓名、职称、单位、研究领域）
- 连线：合作关系（合作次数、合作类型）

### 企业股权网络
- 节点：公司/个人（名称、类型、注册资本）
- 连线：持股关系（持股比例、投资金额）

### 社交关系网络
- 节点：用户（昵称、等级、地区）
- 连线：关注/好友关系（亲密度、互动次数）

## 本地运行

双击 `index.html` 文件即可在浏览器中打开使用，无需服务器。

或者使用本地服务器：
```bash
# 使用 Python
python -m http.server 8000

# 使用 Node.js
npx http-server -p 8000
```

然后访问 `http://localhost:8000`

## 注意事项

1. Excel 文件必须包含 `node` 和 `link` 两个 Sheet（名称包含关键词即可，如 nodes、links、边、关系等）
2. 节点数据和连线数据不能为空
3. 连线的 source 和 target 值必须能在节点中找到对应
4. 导出的 HTML 文件包含所有数据和配置，可直接分享但文件可能较大
